# 日本語の基本知識メモ

このファイルは、AI っぽさの除去だけでなく、美しい・正しい日本語表現に寄せるための補助参照である。厳密な規範集ではなく、推敲時の実務判断に使う。

## 1. 表記体系の前提

- 現代日本語は、漢字、ひらがな、カタカナを併用する
- 必要に応じてローマ字や数字、記号も混在する
- 問題は「どの表記が唯一正しいか」より、文書内で統一されているかにある

見る点:
- 漢字に寄せるか、ひらがなに寄せるか
- 外来語をカタカナにするか、英字のまま残すか
- `Web` / `web` / `ウェブ` のような揺れをどう扱うか
- 句読点を `、。` にするか `，．` にするか

例:
- 悪い例: Webサイトでデータをダウンロードできる。別画面ではウェブサイトからDL可能。
- 改善例: ウェブサイトでデータをダウンロードできる。

## 2. 表記ゆれは品質低下として扱う

表記ゆれは、AI らしさそのものではないが、機械生成感と雑さを強める。

典型例:
- `ユーザー` / `ユーザ`
- `出来る` / `できる`
- `おこなう` / `行う`
- `1つ` / `一つ`
- `Web` / `ウェブ`

判断の原則:
- 固有名詞と UI 表記は原文優先
- それ以外は文書全体で 1 つに揃える
- 迷ったら、その文書の既存慣例に合わせる

## 3. 日本語は品詞感覚が重要

日本語の不自然さは、語彙より品詞選択で生じることが多い。

見る点:
- 名詞化が過剰で、動詞が弱くなっていないか
- 形容動詞や「〜的」の連打で抽象化しすぎていないか
- サ変名詞に依存しすぎていないか

悪い例:
- 本施策は、品質向上の実現に向けた推進を行う。

改善例:
- 本施策で品質を上げる。

補足:
- JustSystems の FAQ と ATOK 解説は、品詞と活用を意識する補助資料として使える

## 4. 直訳調を疑う

英語由来の下書きでは、`enable`, `ensure`, `deliver`, `drive` が日本語で不自然になりやすい。

直訳調の例:
- 意思決定を可能にする
- 価値を提供する
- 成果をドライブする
- 体験を実現する

自然な置き換え:
- 判断しやすくする
- 役立つ
- 成果を押し上げる
- 使いやすくする

## 5. 誤用は「即修正」ではなく文脈判断

誤用とされる表現には、実際には揺れているもの、慣用化したもの、学説が分かれるものがある。

例:
- `現時点ではまだ未定`
- `電車に乗車する`
- `汚名挽回`
- `新年明けましておめでとう`

扱い方:
- 明らかな重言や説明過多は整理する
- 慣用化した表現は、媒体と読者に応じて判断する
- 「誤用らしいから機械的に直す」は避ける

## 6. 助詞と読点はリズムを作る

AI 生成文は、助詞の重なりと読点の打ち方が均一になりやすい。

見る点:
- `の` が連続していないか
- 修飾関係が長くなりすぎていないか
- 読点が多すぎるか、逆に全くないか

悪い例:
- 本機能の管理者向けの設定画面の通知条件の変更。

改善例:
- 本機能では、管理者向け設定画面で通知条件を変更できる。

## 7. 敬語とビジネス文体

丁寧さを上げるほど自然になるとは限らない。AI 生成文では敬語が過剰になりやすい。

悪い例:
- ご確認いただけますと幸甚に存じます。
- ご対応のほど、よろしくお願い申し上げます。

改善例:
- ご確認ください。
- ご対応をお願いします。

原則:
- 目的に足りる敬語で止める
- 丁寧さより明確さを優先する

## 8. 見た目の美しさも品質

日本語では、内容が正しくても見た目が乱れると雑に見える。

見る点:
- 括弧の種類が混在していないか
- 半角と全角が不自然に混在していないか
- 箇条書きの句点有無が揃っているか
- 見出しの粒度が揃っているか
